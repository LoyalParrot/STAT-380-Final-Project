      0,
      1)
    )

Plan and Approach

To answer the research question regarding which maps are most likely to win a vote, I first combined the game data from both players (CODGames_p1_380.csv and CODGames_p2_380.csv) into a single dataset. A significant challenge in this analysis was data quality; specifically, the map names contained numerous spelling errors and inconsistencies (e.g., Ruah for Rush, Collateral Striek for Collateral Strike, and trailing whitespace like Jungle). I addressed this by cleaning the map names in the game data, using the CODMaps.csv file as a reference for correct spellings and mapping identified typos to their canonical names.

For the calculations, I defined a map as a candidate if it appeared in either the Map1 or Map2 columns. To determine the win probability, I analyzed the MapVote and Choice columns. I discovered that the MapVote column consistently formats votes as Winners Vote count to Losers Vote count (e.g., 5 to 0 always, never 0 to 5). Using this, I calculated two metrics:

Strict Wins: The number of times a map earned a strict majority of votes (Vote_Winner > Vote_Loser).

Total Wins: The number of times a map was the final Choice, which includes both strict majorities and instances where the map won via the tie-breaker rule (where Map1 wins if votes are equal). The final probability presented is the Total Win Rate (Total Wins / Candidacies), as this represents the true likelihood of a map being played, but the analysis effectively distinguishes between popularity (strict votes) and systemic advantage (tie-breakers).

def clean_name(name):
    if pd.isna(name):
        return name
    name = str(name).strip()
    
    # Correction dictionary for identified typos
    corrections = {
        'Ruah': 'Rush', 'Riad': 'Raid', 'Raid ': 'Raid',
        'Collateral Striek': 'Collateral Strike', 'Collaterol Strike': 'Collateral Strike',
        'Collaterel Strike': 'Collateral Strike', 'Collateral': 'Collateral Strike',

        'Miami Sstrike': 'Miami Strike', 'Amrada Strike': 'Armada Strike',

        'Jungle ': 'Jungle', 'Rush ': 'Rush', 'Zoo ': 'Zoo', 'Miami ': 'Miami'
    }
    
    if name in corrections:
        return corrections[name]
    
    # Case insensitive match
    for valid in clean_map_names:
        if name.lower() == valid.lower():
            return valid
    return name

# Parse MapVote: Data inspection revealed format is "WinnerVotes to LoserVotes"
def parse_votes(vote_str):
    try:
        parts = str(vote_str).lower().split(' to ')
        return int(parts[0]), int(parts[1])
    except:
        return 0, 0

vote_df[['Vote1', 'Vote2']] = vote_df['MapVote'].apply(lambda x: pd.Series(parse_votes(x)))

Task #4

Research Question:

What is the best predictive accuracy achievable (using an 80/20 validation split) for Logistic Regression, Random Forest, and Support Vector Machine (SVM) when predicting whether a player wins their Call of Duty match using only: Eliminations, Deaths, Damage, DidPlayerVote (Yes/No), PrimaryWeapon, XPType, and GameType?

Decision Made / Data Setup (Response + Features)

The original Result column is stored as a score string (ex: 100-97). I converted this to a binary outcome by splitting the two scores and defining Win when the first score is larger and Loss otherwise.

As mentioned, only the predictors listed in the research question are used. I restricted predictors to exactly the variables listed in the question to keep the comparison fair and also to eliminate variables which I personally thought would be uninformative about a players chance of winning like the date, map (since both teams have to use the same map), etc. I used Ishans earlier dataframe of CodGamesAllPlayersCleaned because it was previously cleaned and adjusted to make sure all games have the necessary info and all players participated. All categorical variables were converted to factors including DidPlayerVote, PrimaryWeapon, XPType, and GameType. Lastly, I did not factor out partial games as they were still a large part of the dataset and I think it was important to consider them, as many games I have played were partial/not completed and the my personal goal with this research question is to have the best predictive accuracy for any game not just full/partial.

Going back to variable selection here is my reasoning for the 7 chosen variables: - Eliminations: If a player has more eliminations it seems like this would benefit their chance of winning by increasing their teams score more and quicker than their oponents - Deaths: If a player has more deaths it seems like they are getting eliminated more than they are eliminating opponents likely decreasing chances of winning - Damage: While eliminations and damage are likely very correlated, I think damage does a good job also representing a players impact as sometimes another player may help eliminate someone even though you did a ton of damage to them and thus that is why including damage likely will improve the models - DidPlayerVote: I think including this may seem random, but I think that if a player is voting they are more likely to have an opinion on the map due to them feeling like it may increase their chance of winning. Thus by voting and increasing the chances of getting a favorable map to them I think this could help increase the models predictive accuracy - Primary Weapon: I think some weapons definitely help players more than others and could benefit certain playstyles leading to higher chance of winning - XPType: If a player has a higher xp boost during a game they may be incentivized to play better and harder and thus as a result higher win rates - GameType: While I am not familiar with the different gametypes, Im sure they could definitely be beneficial to some players and not others based on gamestyle so that is why it was included in the model

To ensure a fair comparison between methods, I used the same 80/20 training/validation split (seed = 123) for every model, and compared performance using validation accuracy (confusion matrix + accuracy). I also reported AUC when the method provides class probabilities, as a secondary metric for how well the model separates Win vs Loss across thresholds.

scale_with_train <- function(x_train, x_valid) {
  mu <- mean(x_train, na.rm = TRUE)

  if (s == 0) s <- 1
